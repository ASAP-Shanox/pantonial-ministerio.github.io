Creating a Personal Blog Using HTML and CSS

Creating a personal blog using only HTML and CSS will result in a static website. This means that every blog post will be a separate HTML file, and you'll need to manually create and update each one. There won't be dynamic features like user comments, search functionality, or a database to manage posts.

However, it's a great way to learn the fundamentals of web structure and styling. Here's a step-by-step guide:

1. Planning Your Blog Structure:

  • Homepage (`index.html`): This will be the main page, listing your latest blog posts with brief summaries and links to the full posts.
  • Individual Blog Post Pages (`post1.html`, `post2.html`, etc.): Each blog post will have its own dedicated HTML file.
  • CSS File (`style.css`): This file will contain all the styling rules for your blog.

2. Setting Up Your Project Folder:

Create a folder on your computer to organize your blog files. Inside this folder, create:

  • `index.html` (for your homepage)
  • A subfolder named `posts` (to store individual blog post HTML files)
  • `style.css` (for your CSS)
  • An optional folder named `images` (to store any images you'll use)

3. Creating the Homepage (`index.html`):

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>My Personal Blog</title>
  <link rel="stylesheet" href="style.css">
</head>
<body>
  <header>
    <h1>My Awesome Blog</h1>
    <p>Sharing my thoughts and experiences.</p>
  </header>
  <main>
    <section class="latest-posts">
      <h2>Latest Posts</h2>
      <article class="post-summary">
        <h3><a href="posts/first-post.html">My First Blog Post</a></h3>
        <p class="post-meta">Published on April 12, 2025</p>
        <p>A brief summary of my first blog post. This might include the first few sentences to give readers an idea of what it's about. <a href="posts/first-post.html">Read more</a></p>
      </article>
      <article class="post-summary">
        <h3><a href="posts/second-post.html">Another Interesting Topic</a></h3>
        <p class="post-meta">Published on April 10, 2025</p>
        <p>A short preview of my second blog entry. Engage your readers with a compelling snippet. <a href="posts/second-post.html">Read more</a></p>
      </article>
    </section>
  </main>
  <footer>
    <p>© 2025 My Personal Blog</p>
  </footer>
</body>
</html>

4. Creating Individual Blog Post Pages (e.g., `posts/first-post.html`):

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">

</head>
<body>
  <header>
    <h1>My Awesome Blog</h1>

  </header>

      <h2>My First Blog Post</h2>
      <p class="post-meta">Published on April 12, 2025</p>

      <p>I can also include multiple paragraphs, lists:</p>
      <ul>

        <li>Point 2</li>
        <li>Point 3</li>
      </ul>

      <img src="../images/example.jpg" alt="An example image">
      <h3>A Subheading</h3>
      <p>More detailed information under a subheading.</p>
    </article>
  </main>
  <footer>
    <p>© 2025 My Personal Blog</p>
  </footer>
</body>
</html>

6. Adding New Blog Posts:

To add a new blog post:

  1. Create a new HTML file in the `posts` folder (e.g., `posts/third-post.html`).
  2. Copy the basic structure from an existing post file and modify the content, title, and publication date.
  3. On your homepage (`index.html`), add a new `
    ` section linking to your new post file.

Limitations of a Static Blog:

  • Manual Updates: Adding and editing posts requires manually creating and modifying HTML files.
  • No Dynamic Features: You won't have features like comments, search, categories, or tags without server-side scripting and a database.
  • Scalability: Managing a large number of posts can become cumbersome.

Moving Beyond Static:

If you want a more dynamic and feature-rich blog, you'll need to explore:

  • Static Site Generators (SSGs): Tools like Jekyll, Hugo, and Next.js allow you to write blog posts in Markdown or other formats and automatically generate static HTML files. They often offer features like themes, categories, and basic organization.
  • Content Management Systems (CMS): Platforms like WordPress, Drupal, and Ghost provide a user-friendly interface for creating, managing, and publishing blog content, along with dynamic features and databases.

In Conclusion:

Building a personal blog with just HTML and CSS is a great learning exercise to understand the fundamental building blocks of the web. While it will be a static site with manual updates, it can still serve as a simple platform to share your thoughts. As your needs grow, you can explore more dynamic solutions. Remember to keep practicing and experimenting!

Learning HTML & CSS Basics

December 14, 2024 • Web Development

Our first steps into web development at Cebu Eastern Colleges...

HTML forms the foundation of every webpage. Think of it as the blueprint of a house, defining the rooms, doors, and windows. Here's a more detailed look:

HTML Key Concepts

Elements

The building blocks of HTML documents. They are defined by tags.

Tags

Keywords enclosed in angle brackets (<>). Most tags come in pairs: an opening tag (e.g., <div>) and a closing tag (e.g., </div>), with content in between. Some tags are self-closing (e.g., <br>, <hr>, <img src=" ">).

<!-- Example of paired tags -->
<p>This is a paragraph.</p>

<!-- Example of self-closing tag -->
<img src="image.jpg" alt="Description">

Attributes

Provide additional information about HTML elements. They are specified within the opening tag and consist of a name and a value (e.g., <img src="image.jpg" alt="My Image">). Common attributes include id, class, style, src, href, alt, etc.

Document Structure

A basic HTML document has a specific structure:

<!DOCTYPE html>
<html>
  <head>
    <title>Page Title</title>
  </head>
  <body>
    <h1>My First Heading</h1>
    <p>My first paragraph.</p>
  </body>
</html>
  • <!DOCTYPE html>: Declares the document type as HTML5.
  • <html>: The root element of the HTML page.
  • <head>: Contains meta-information about the HTML document, such as the title, character set, links to CSS files, and scripts. This information is generally not displayed on the page itself.
  • <title>: Specifies the title that appears in the browser tab or window title bar.
  • <meta>: Provides metadata about the HTML document (e.g., character encoding, viewport settings, keywords).
  • <body>: Contains the visible page content (text, images, links, etc.).

Semantic HTML

Using HTML tags according to their meaning. For example, using <article> for a self-contained composition, <nav> for navigation menus, and <aside> for content related to the main content. This improves SEO, accessibility, and code readability.

Pro Tip: Always prefer semantic HTML elements over generic <div> elements when possible. It makes your code more meaningful and accessible.

Common HTML Elements

  • Text Formatting: <h1> to <h6> (headings), <p> (paragraph), <strong> (strong emphasis), <em> (emphasis), <span> (inline container).
  • Lists: <ul> (unordered list), <ol> (ordered list), <li> (list item), <dl> (description list), <dt> (description term), <dd> (description description).
  • Links: <a> (anchor tag) for creating hyperlinks using the href attribute.
  • Images: <img> for embedding images using the src attribute.
  • Tables: <table>, <tr> (table row), <th> (table header), <td> (table data cell).
  • Forms: <form>, <input>, <textarea>, <button>, <select>, <option>, etc., for creating interactive forms.
  • Divisions and Spans: <div> (block-level container), <span> (inline container) used for structuring and styling content.
  • Semantic Elements: <article>, <aside>, <nav>, <header>, <footer>, <section>, etc., for providing meaning to the structure of the document.

Exploring CSS in Detail

CSS is the styling language that controls the visual presentation of HTML elements. It's like the interior designer and decorator of our webpage house, determining the colors, fonts, layout, and overall look and feel.

Key CSS Concepts

Selectors

Patterns used to select the HTML elements you want to style. Common selectors include:

  • Element Selectors: Select all elements of a specific type (e.g., p, h1, div).
  • ID Selectors: Select a single element based on its unique id attribute (e.g., #my-heading).
  • Class Selectors: Select all elements with a specific class attribute (e.g., .button).
  • Attribute Selectors: Select elements based on their attributes or attribute values (e.g., [type="text"], a[href="example"]).
  • Pseudo-classes: Select elements based on their state or position (e.g., :hover, :first-child, :nth-child(even)).
  • Pseudo-elements: Creates virtual elements within an existing element for styling specific parts (e.g., ::before, ::after).
/* Element selector */
p { color: blue; }

/* Class selector */
.highlight { background-color: yellow; }

/* ID selector */
#main-header { font-size: 2rem; }

/* Attribute selector */
input[type="text"] { border: 1px solid #ccc; }

/* Pseudo-class */
a:hover { text-decoration: underline; }

/* Pseudo-element */
p::first-line { font-weight: bold; }

Properties and Values

  • Properties: Define the visual characteristics you want to change (e.g., color, font-size, background-color, width, height).
  • Values: Specify the settings for CSS properties (e.g., red, 16px, lightblue, 100%, auto).

Ruleset (or Rule)

A combination of a selector and one or more declarations (property-value pairs) enclosed in curly braces {}.

Cascading

The "C" in CSS stands for Cascading. This refers to how styles are applied when multiple rules target the same element. Styles from different sources (browser defaults, external stylesheets, internal styles, inline styles) are applied based on specificity and order.

Specificity

Determines which CSS rule takes precedence when multiple rules apply to the same element. More specific selectors override less specific ones (e.g., an ID selector is more specific than a class selector, which is more specific than an element selector).

Inheritance

Some CSS properties are inherited from parent elements to their children (e.g., font-family, color). However, not all properties are inherited (e.g., border, padding, margin).

Box Model

A fundamental concept in CSS that describes the rectangular boxes that are generated for HTML elements. It consists of:

  • Content: The actual content of the element (text, images, etc.).
  • Padding: Space around the content, inside the border.
  • Border: A line surrounding the padding and content.
  • Margin: Space outside the border, separating the element from other elements.
Remember: When calculating the total width of an element, you need to add content + padding + border. Margins are not included in the element's width.

Layout Models

CSS provides different layout models to arrange elements on a page:

  • Normal Flow: Elements are laid out in the order they appear in the HTML source. Block-level elements take up the full width available and stack vertically, while inline elements flow horizontally within their parent.
  • Flexbox (Flexible Box Layout): A powerful one-dimensional layout model for arranging items in rows or columns. It's excellent for aligning and distributing space among items within a container.
  • Grid Layout: A two-dimensional layout model for creating complex, grid-based layouts with rows and columns. It offers more control over positioning elements in both dimensions.
  • Positioning: Allows you to control the placement of elements on the page using the position property (e.g., static, relative, absolute, fixed, sticky).
  • Float: Used to position and wrap text around elements (though often superseded by Flexbox and Grid for more complex layouts).

Responsiveness

Designing websites that adapt and display well on different screen sizes and devices. Techniques include:

  • Media Queries: Allow you to apply different CSS rules based on screen width, height, device orientation, etc. (@media screen and (max-width: 768px) { ... }).
  • Fluid Layouts: Using percentage-based widths and heights instead of fixed pixel values.
  • Flexible Images and Videos: Ensuring media scales appropriately.
  • Mobile-First Approach: Designing for smaller screens first and then progressively enhancing for larger screens.
/* Mobile-first media query */
.container {
  width: 100%;
  padding: 10px;
}

@media (min-width: 768px) {
  .container {
    width: 750px;
    margin: 0 auto;
  }
}

In Summary

HTML provides the structure and meaning of your web content, while CSS handles the visual presentation and layout. Mastering both is crucial for creating well-structured, visually appealing, and user-friendly websites. They work together harmoniously, with HTML defining what the content is and CSS defining how it looks. Understanding their individual roles and how they interact is the key to effective web development.

As IT students at Cebu Eastern Colleges, we've found that practicing these concepts by building small projects is the best way to solidify our understanding. Start with simple pages and gradually add more complex elements and styling as you become more comfortable with the basics.

Balancing School & Projects

February 15, 2025 • Student Life

How we manage our time between academics and personal projects...

Balancing school and projects as an IT student requires a strategic approach to time management, organization, and self-care. Here's a breakdown of how to achieve this balance:

1. Time Management is Key

Create a Master Schedule

Use a calendar or planner (digital or physical) to map out all your commitments: classes, assignments, exams, project deadlines, work (if applicable), social events, and personal time. Color-coding can help distinguish between different types of activities.

Prioritize Tasks

Understand the difference between urgent and important tasks. Focus on high-priority items first, such as assignments with earlier deadlines or those that are worth more marks. Use methods like the Eisenhower Matrix (urgent/important) to help.

Pro Tip: We use Google Calendar with color coding - blue for classes, red for deadlines, green for personal time. This visual system helps us see our week at a glance.

Time Blocking

Allocate specific blocks of time for studying, working on projects, and other activities. Treat these blocks like appointments and stick to them as much as possible.

Break Down Large Tasks

Divide big projects into smaller, more manageable steps with their own mini-deadlines. This makes the overall task less daunting and helps you track progress.

Start Early

Don't wait until the last minute to begin assignments or projects. Starting early reduces stress and allows for unexpected delays.

2. Organization is Crucial

  • Keep Everything in One Place: Whether it's a physical binder or a digital system, organize your notes, assignments, project files, and deadlines in a way that's easy to access.
  • Use Project Management Tools: We recommend Trello for visual task management or Notion for all-in-one organization.
  • Note-Taking Strategies: Develop a consistent system for class notes. We use the Cornell method for important lectures.
  • Maintain a Clean Workspace: A clutter-free environment helps maintain focus. We do a 5-minute cleanup at the end of each study session.

3. Project-Specific Strategies

  • Understand Requirements: Before starting, make sure you fully understand what's expected. We create a checklist of deliverables.
  • Create a Project Plan: Break the project into phases with milestones. For coding projects, we use GitHub Projects to track progress.
  • Regular Communication: For group projects, set weekly check-ins. We use Discord for quick communication between meetings.
  • Version Control: Essential for coding projects. We commit code daily with descriptive messages.
Our Workflow: We dedicate Wednesday evenings and Sunday afternoons specifically for project work. This consistent schedule helps us make steady progress.

4. Self-Care is Essential

You can't pour from an empty cup. Here's what works for us:

  • Sleep: We aim for 7-8 hours nightly, even during busy periods.
  • Nutrition: Meal prepping on Sundays ensures we eat well during the week.
  • Exercise: Even short walks between study sessions make a difference.
  • Downtime: We schedule guilt-free breaks for gaming or watching shows.

5. Leverage Resources

Don't try to do everything alone:

  • Form study groups with classmates
  • Visit professors during office hours
  • Use campus resources like tutoring centers
  • Connect with upperclassmen for advice

By implementing these strategies, we've been able to maintain good grades while working on personal projects that enhance our skills. Remember that balance looks different for everyone - experiment to find what works best for you!

Building Our First Website

March 06, 2025 • Projects

The story behind creating this blog and what we learned...

That's exciting! Building your first website with HTML is a fundamental step into the world of web development. Here's a breakdown of the essential steps and concepts to get you started:

1. Setting Up Your Environment:

Text Editor: You'll need a plain text editor to write your HTML code. Here are some popular (and often free) options:

  • Beginner-Friendly: Notepad (Windows), TextEdit (Mac - make sure to save as plain text), Sublime Text, VS Code (Visual Studio Code), Atom.
  • Recommended for Beginners: VS Code is highly recommended due to its helpful features like syntax highlighting, auto-completion, and integrated terminal.

Web Browser: You'll need a web browser (like Chrome, Firefox, Safari, Edge) to view the HTML files you create.

2. Creating Your First HTML File:

  1. Open your text editor.
  2. Write the basic HTML structure: Every HTML document starts with a specific structure. Copy and paste this into your editor:
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>My First Website</title>
</head>
<body>
  <h1>Hello, World!</h1>
  <p>This is my first website built with HTML.</p>
</body>
</html>
  1. Save the file:
  • Go to `File` > `Save As...`
  • Choose a location on your computer where you want to save the file.
  • Crucially, name the file with the `.html` extension. For your main page, it's common to name it `index.html`.
  • Make sure the "Save as type" or "Format" is set to "All Files" or "Plain Text" (depending on your editor).

3. Understanding the Basic HTML Structure:

  • <!DOCTYPE html>: This declaration defines the document type to be HTML5. It's the very first thing in your HTML document.
  • <html lang="en">: This is the root element of the HTML page. The `lang="en"` attribute specifies the language of the document as English.
  • <head>: This section contains meta-information about the HTML document, which is not displayed on the page itself.
    • <meta charset="UTF-8">: Specifies the character encoding for the document, which is usually UTF-8 for broad character support.
    • <meta name="viewport" content="width=device-width, initial-scale=1.0">: Configures the viewport for responsive design, ensuring your website looks good on different screen sizes.
    • <title>My First Website</title>: Sets the title that appears in the browser tab or window title bar.
  • <body>: This section contains all the visible content of your webpage.
    • <h1>Hello, World!</h1>: This is a level 1 heading. HTML has six levels of headings (`<h1>` to `<h6>`).
    • <p>This is my first website built with HTML.</p>: This is a paragraph of text.

4. Viewing Your Website:

  1. Locate the `index.html` file you saved on your computer.
  2. Double-click the file. This will open it in your default web browser.

You should now see a simple webpage with the heading "Hello, World!" and the paragraph "This is my first website built with HTML."

5. Adding More Content and Structure:

Now, let's add some more common HTML elements to build out your website:

  • Headings: Use `<h1>` to `<h6>` for different levels of headings.
  • Paragraphs: Use `<p>` for blocks of text.
  • Links: Use the `<a>` (anchor) tag to create hyperlinks. The `href` attribute specifies the link destination.
<a href="https://www.google.com" target="_blank">Visit Google</a>
  • href: The URL of the page you want to link to.
  • target="_blank": Opens the link in a new browser tab.
  • Images: Use the `<img>` tag to embed images. The `src` attribute specifies the path to the image file, and the `alt` attribute provides alternative text if the image cannot be displayed.
<img src="your-image.jpg" alt="Description of the image">
  • Make sure `your-image.jpg` is in the same folder as your `index.html` file, or provide the correct path to the image.
  • Lists:
    • Unordered List (`<ul>`): Creates a bulleted list.
    • Ordered List (`<ol>`): Creates a numbered list.
    • List Items (`<li>`): Used inside `<ul>` or `<ol>` to represent individual items.
<ul>
  <li>Item 1</li>
  <li>Item 2</li>
  <li>Item 3</li>
</ul>

<ol>
  <li>First step</li>
  <li>Second step</li>
  <li>Third step</li>
</ol>
  • Divisions (`<div>`): A generic container element used to group other HTML elements for styling or layout purposes.
  • Spans (`<span>`): An inline container element used to group small pieces of text or other inline elements for styling.

6. Next Steps:

  • Experiment: Try adding different HTML elements and see how they are displayed in your browser.
  • Learn More Tags: Explore other HTML tags and their attributes. Refer to online documentation like the MDN Web Docs (Mozilla Developer Network).
  • Introduction to CSS: Once you have a basic understanding of HTML, the next crucial step is to learn CSS (Cascading Style Sheets). CSS is used to style the appearance of your HTML elements (colors, fonts, layout, etc.). You can link external CSS files to your HTML or embed CSS directly within the `<style>` tags in the `<head>` section.

Example of Adding Basic CSS (within `<style>` tags in the `<head>`):

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>My First Website with Basic Styling</title>
  <style>
    body {
      font-family: sans-serif;
      background-color: #f0f0f0;
      margin: 0;
      padding: 20px;
    }
    h1 {
      color: blue;
      text-align: center;
    }
    p {
      line-height: 1.5;
    }
  </style>
</head>
<body>
  <h1>Welcome to My Styled Website!</h1>
  <p>This paragraph has some basic styling applied using CSS.</p>
  <a href="https://www.example.com" target="_blank">Check out Example.com</a>
</body>
</html>

Building your first website is a journey. Start with the basics, experiment, and gradually learn more complex concepts. Don't be afraid to make mistakes – they are part of the learning process! Good luck!
